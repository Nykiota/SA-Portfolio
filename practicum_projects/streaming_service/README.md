## Стриминговый сервис

### Задача:
Провести анализ измененной архитектуры стримингового сервиса (переход на REST API, документация в Swagger) и на основе интервью с заказчиком формализовать требования для постановки задач разработчикам на доработку API.

### Шаги для решения задачи::

- *Анализ требований*: Провел анализ интервью с представителем заказчика для выявления и верификации бизнес-требований.

- *Проектирование и анализ системы*:

* Создал [Диаграмму потоков данных (DFD)](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/streaming_service/Seq-Diag.drawio) для визуализации обновленной архитектуры и взаимодействий компонентов.
* Разработал Диаграмму последовательности для детализации сценариев работы новой функции, обеспечив однозначное понимание логики разработчиками.
* Выполнил [анализ и проверку XML-схемы](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/streaming_service/XML-check.pdf) на соответствие новым требованиям.

- *Тестирование и формализация задач*:

Протестировал существующее API через Swagger для оценки его текущего состояния и выявления несоответствий.
На основе всего комплекса работ подготовил и зафиксировал [детализированный тикет на разработку](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/streaming_service/Ticket-for-Devs.pdf) новой операции API.

### Результат:
Комплексный анализ и проектная документация (DFD, диаграмма последовательности, протокол проверки API), на основе которых в бэклог разработки была добавлена четкая, технически обоснованная задача на реализацию новой функциональности. Это обеспечило прозрачность планирования и позволило команде приступить к работе без дополнительных уточнений.
