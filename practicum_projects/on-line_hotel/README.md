## Cайт для отеля 


### Задача:
Обеспечить успешный запуск веб-сервиса бронирования капсульных отелей путем проведения финального контроля качества и создания комплекта приемочной документации. Ключевая цель — подтвердить готовность функционала к передаче пользователям и предоставить Заказчику формальные основания для приемки работ.

### Шаги для решения задачи:

Работы были выполнены в два ключевых этапа

- *Функциональное тестирование и стабилизация*: Мы провели целенаправленное [тестирование](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/on-line_hotel/test-plan.pdf) критически важных для бизнеса модулей (регистрация, ЛК, бронирование). Выявленные дефекты были зафиксированы, переданы разработчикам и успешно исправлены, что привело продукт в стабильное, рабочее состояние.

- *Разработка приемочной документации*: После стабилизации функционала мы подготовили итоговый комплект документов:

**[Программу и методику испытаний (ПМИ)](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/on-line_hotel/program-and-test-methodology.pdf)** — формальный документ для Заказчика, позволяющий ему провести объективную проверку соответствия продукта всем требованиям.

**[Руководство пользователя](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/on-line_hotel/user-manual.pdf)** — итоговый артефакт, который служит как для приемки (как доказательство завершенности и продуманности продукта), так и для будущей эксплуатации.

### Результат:
Заказчик получил полностью готовый к запуску веб-сервис, прошедший полный цикл тестирования и стабилизации, а также полный пакет документации для проведения формальной приемки. Это позволило перейти к финальной стадии проекта — передаче продукта в эксплуатацию — с минимальными рисками и четкими критериями приемки.
