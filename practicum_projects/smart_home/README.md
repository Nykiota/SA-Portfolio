## Приложение Умный Дом


### Задача:
Спроектировать и подготовить к разработке фирменное мобильное приложение для управления инновационной умной лампочкой с уникальной функцией включения/выключения режима энергосбережения, которая не поддерживается в сторонних решениях для умного дома.

### Шаги для решения задачи:

Выполнил полный цикл предпроектного анализа и проектирования:

- Провели декомпозицию функциональных требований через [Карту пользовательских историй](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/smart_home/User%20Story%20MVP.pdf)
- Спроектировал модель хранения данных с помощью [ERD (Entity Relationship Diagram)](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/smart_home/ERD.drawio)
- Определили границы системы и информационные потоки через DFD (Data Flow Diagram) [конекстную](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/smart_home/DFD-context.drawio) и [логическую](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/smart_home/DFD-context.drawio)
- Формализовал структуру данных в [Словаре данных](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/smart_home/Glossary%20data.pdf)
- Визуализировал пользовательский интерфейс через [Интерактивные прототипы](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/smart_home/Prototypes.pdf)
- Разработал [Программу и методику испытаний (ПМИ)](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/smart_home/test-program-and-methodology.pdf) для последующего тестирования 

### Результат:
Полный пакет проектной документации, включающий модели данных, архитектурные диаграммы и спецификации, который позволил:

- Четко формализовать уникальное ценностное предложение продукта
- Обеспечить команду разработки исчерпывающими требованиями
- Создать основу для тестирования и приемки продукта
- Ускорить переход к стадии активной разработки приложения
