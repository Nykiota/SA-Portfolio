## Сеть клиник

### Задача:
Провести полный цикл анализа и проектирования мобильного приложения для сети медицинских клиник с целью перевода устаревшего процесса записи по телефону в современный цифровой формат. Ключевые цели: повышение удобства для пациентов (целевая аудитория 30+), оптимизация нагрузки на кол-центр и создание платформы для будущего роста (включая запуск новой услуги "Чекап").

### Шаги для решения задачи:
Для решения задачи был применен комплексный аналитический подход:

- *Анализ и оптимизация бизнес-процесса*: Проведены интервью со стейкхолдерами, смоделирован и проанализирован [процесс "как есть" (AS-IS)](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/BPMN-AS-IS.drawio)  в BPMN и [State Machine Diagram](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/%20UML_SMD.drawio), на основе чего спроектирован оптимальный процесс ["как будет" (TO-BE)](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/BPMN-TO-BE.drawio) с максимальной автоматизацией.

- *Формализация требований и проектирование системы*: На основе гипотез и обсуждений с командой (разработка, дизайн) созданы персоны, [карта пользовательских историй](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/User%20Story%20MVP.png) и определено MVP. Разработаны диаграммы потоков данных (DFD)[контекстная](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/DFD-context.drawio) и [логическая](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/DFD-logic.drawio) для четкого определения границ системы и взаимодействий.

- *Проектирование пользовательского опыта и функциональности*: Создана раскадровка (Storyboard) и [прототипы интерфейсов](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/Prototypes_1.pdf), визуализирующие ключевые сценарии записи к врачу и управления визитами. Дополнительно спроектирована и интегрирована в MVP новая бизнес-функция — автоматизированный подбор ["Чекапа"](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/User-Story_MVP.pdf) на основе данных пациента со своими [экранами интерфейса](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/clinic/Prototypes_2.pdf).

- *Определение архитектурных ограничений*: Зафиксированы ключевые решения по безопасности и архитектуре: упрощенная авторизация по SMS для доступа к ЛК, публичный доступ к каталогу врачей, правила привязки врачей к филиалам и уникальности их профилей.

### Результат:
Команда получила исчерпывающий пакет проектной документации, включающий:

- Согласованные модели процессов (AS-IS/TO-BE).
- Приоритизированный бэклог в формате пользовательских историй и четко определенное MVP.
- Прототипы интерфейсов и раскадровки, утвержденные заказчиком.
- Детальное техническое задание на функцию "Чекап".
- Архитектурные решения (DFD) и бизнес-правила.

Этот результат позволил без задержек и недопонимания перейти к этапу разработки, имея на руках полное видение продукта, его границ и приоритетов.
