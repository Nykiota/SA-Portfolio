## Сайт-агрегатор для покупки авиабилетов

### Задача:
Формализовать и визуализировать сложный процесс взаимодействия между клиентом, сайтом-агрегатором, тремя авиакомпаниями (А,В,С)  и платежной экосистемой («Сам сервис», «Банк-эквайер», «Банк-эмитент») для обеспечения корректной реализации системы поиска, бронирования и оплаты авиабилетов.

### Шаги для решения задачи:
Проанализировал техническое задание и спроектировал [UML-диаграмму последовательности](https://github.com/Nykiota/SA-Portfolio/blob/main/practicum_projects/flight_ticket_aggregator/Sequence-Diagram.drawio), которая детально описывает:

- Параллельный запрос данных к API трёх авиакомпаний
- Процесс бронирования билета с ограничением времени (15 минут)
- Многошаговую интеграцию с платежной системой
- Полный цикл подтверждения оплаты и выписки билета
- Все взаимодействия в REST-архитектуре

### Результат:
Создана четкая и исчерпывающая диаграмма последовательности, которая стала ключевым артефактом для:

- Однозначного понимания логики взаимодействия между всеми компонентами системы
- Корректной реализации интеграций с внешними API
- Снижения рисков misunderstandigs в команде разработки
- Ускорения процесса разработки сложного распределенного сценария
